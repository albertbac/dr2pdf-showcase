# Security Boundary

This repository is a public-safe showcase for dr2pdf. It is documentation-only and must never be treated as the product repository.

## Allowed public material
- product framing at a high level
- public-safe workflow summaries
- architecture diagrams without private runtime detail
- manual TODOs for synthetic screenshots and validated metrics

## Never publish
- production source code
- real records or screenshots
- private access material
- copied operating logic
- deployment internals
- raw data traces
- institution-, user-, or client-identifying detail

## Second-pass review rule
Before every push, confirm that:
- the repository still matches the minimum public showcase file set
- manual evidence is marked as TODO until safely produced
- no new app-private detail was copied from the private workspace
- GitHub description and topics remain aligned with the portfolio taxonomy

If material doubt remains, block publication and rewrite the affected file before pushing.
